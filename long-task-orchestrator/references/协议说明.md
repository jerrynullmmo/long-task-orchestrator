# 协议说明

## 状态文件

长任务最小闭环由 5 类文件组成：

| 文件 | 作用 |
|---|---|
| 当前状态 | 记录项目、阶段、状态版本和下一步 |
| 调度记录 | 记录主线程为什么分派、分派给谁、是否例外 |
| 任务包 | 给执行单元的唯一授权与验收合同 |
| 回执 | 执行单元完成后写回的机器可读证据 |
| 监控状态 | 主线程或自动化检查回执是否出现 |

## 恢复交接字段

上下文压缩后的摘要不是完整现场。每次恢复、重大返工或高风险产物放行前，`当前状态` 必须保留 `恢复前检查`：

| 字段 | 作用 |
|---|---|
| 原始目标 | 防止中途任务把最终交付带偏 |
| 禁止路径 | 防止执行单元触碰旧边界或高风险路径 |
| 失败记录 | 防止重复执行已否方案 |
| 待核事实 | 防止把存疑判断写成确定事实 |
| 下一步闸门 | 防止自然语言完成声明绕过回执、校验或人工确认 |

任一字段为空时，主线程不得继续分派正式执行单元；应先补齐 `当前状态` 或重新建立任务包。

## 主线程规则

- 主线程不直接做高风险产物。
- 主线程不持续轮询执行单元聊天上下文。
- 主线程只用任务包、回执、监控状态和必要脚本结果放行。
- 主线程可以创建新任务包，但不能让执行单元自行扩大任务范围。

## 线程监控规则

独立线程创建后，主线程允许做一次短暂拉起确认。该动作只确认线程存在、可读和可登记，不确认是否已经产出。

后续监控必须满足：

- 任务包 `动作策略.允许监控动作` 包含 `confirm_thread_started`、`register_thread_heartbeat`、`heartbeat_check_status`。
- 任务包 `动作策略.永久阻断动作` 包含 `manual_short_poll_thread_output`。
- 监控状态 `心跳周期分钟` 固定为 `15`。
- 监控状态 `心跳创建状态` 为 `已创建`。
- 监控状态 `短轮询禁止` 为 `true`。
- `进度判定来源` 只能来自心跳、执行单元主动回执、回执监控文件或用户明确要求的人工介入。
- 心跳创建失败时暂停调度，不得改用短等待、线程预览、文件存在性检查或人工轮询补救。

## 执行单元规则

- 只做任务包内的目标。
- 只读取任务包要求的文件。
- 只写允许写入路径和授权产物路径。
- 完成后写回执，不自行进入下一阶段。
- 遇到阻塞时写明阻塞问题，不编造完成状态。

## 高风险产物

以下产物默认高风险：

- 应用入口代码、配置、迁移脚本、构建脚本。
- 对外可见文档、发布物、成片、部署配置。
- 会触发外部系统、账号、支付、删除或发布的脚本。

高风险产物必须走 `独立线程` 或用户确认的等价隔离载体。
