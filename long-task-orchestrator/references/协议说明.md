# 协议说明

## 状态文件

长任务最小闭环由 5 类文件组成：

| 文件 | 作用 |
|---|---|
| 当前状态 | 记录项目、阶段、状态版本和下一步 |
| 调度记录 | 记录主线程为什么分派、分派给谁、是否例外 |
| 任务包 | 给执行单元的唯一授权与验收合同 |
| 回执 | 执行单元完成后写回的机器可读证据 |
| 监控状态 | 主线程或自动化检查回执是否出现 |

## 主线程规则

- 主线程不直接做高风险产物。
- 主线程不持续轮询执行单元聊天上下文。
- 主线程只用任务包、回执、监控状态和必要脚本结果放行。
- 主线程可以创建新任务包，但不能让执行单元自行扩大任务范围。

## 执行单元规则

- 只做任务包内的目标。
- 只读取任务包要求的文件。
- 只写允许写入路径和授权产物路径。
- 完成后写回执，不自行进入下一阶段。
- 遇到阻塞时写明阻塞问题，不编造完成状态。

## 高风险产物

以下产物默认高风险：

- 应用入口代码、配置、迁移脚本、构建脚本。
- 对外可见文档、发布物、成片、部署配置。
- 会触发外部系统、账号、支付、删除或发布的脚本。

高风险产物必须走 `独立线程` 或用户确认的等价隔离载体。
