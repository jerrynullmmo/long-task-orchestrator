# Codex 适配

## 推荐载体

- 主线程：当前 Codex 对话。
- `subagent`：使用 Codex 暴露的子代理工具，适合只读复核、证据采集和挑刺。
- `独立线程`：使用 Codex 新线程或新窗口，适合高风险产物和正式阶段产物。
- `命令任务`：使用本地终端命令，适合可重复脚本、测试和构建。

## 主线程下发新线程任务

新线程任务提示应包含：

1. 项目目录。
2. 任务包完整路径和内容。
3. 允许写入路径。
4. 禁止触碰路径。
5. 回执路径。
6. 完成后必须停止并等待主线程处理。

## 监控策略

Codex 首选心跳自动化检查回执文件。没有自动化能力时，主线程可以用文件事件监听脚本或定期读取监控状态文件，但不要持续读取执行线程聊天上下文。

## 放行策略

主线程必须在当前线程运行校验脚本，并根据业务验收标准决定是否推进。执行线程回执通过不等于自动放行。
