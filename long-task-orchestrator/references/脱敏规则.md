# 脱敏规则

## 禁止进入开源仓库

- 组织内部路径、个人主目录、内部项目名。
- 生产线程号、窗口号、自动化编号。
- 外部联系方式、账号凭据、报价、合同内容。
- 内部业务脚本、未公开方法论和运营数据。

## 替换方式

| 敏感类型 | 替换方式 |
|---|---|
| 组织路径 | `/workspace/demo-project` |
| 生产线程号 | `演示执行单元-001` |
| 业务项目名 | `示例项目` |
| 高风险产物 | `src/output.md` 或 `dist/report.html` |
| 内部标准 | `docs/spec.md` |

## 自查

不要把私有关键词或私有正则写入仓库。发布前在仓库根目录创建本地文件 `.redaction-keywords`，每行写一个只在本机使用的敏感片段；需要正则时给该行加 `re:` 前缀。该文件已加入 `.gitignore`。

然后运行：

```bash
python3 long-task-orchestrator/scripts/check_redaction.py --keywords .redaction-keywords .
```

如果命中公开示例、模板或文档，必须改成脱敏占位。需要长期复用的私有关键词只放在本地清单，不提交到开源仓库。
